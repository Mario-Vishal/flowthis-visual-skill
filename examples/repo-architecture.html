Skill Recipes

The unique skill is not a rigid visual theme. It is a set of reusable decisions that make any generated visual clearer, safer, and more shareable.

◎

Visual brief

Every artifact starts by naming source type, audience, confidence, reader job, and what to inspect first.

▤

Explorer-detail pair

Repos and long documents get compact navigation plus a summary pane, not a pile of disconnected cards.

⇢

Focus map

One primary diagram shows the relationship between actors, services, storage, boundaries, and risks.

≣

Evidence rail

Important claims point to files, sections, figures, comments, or are clearly labeled as inference.

◇

Decision ledger

Architecture choices are stored as decision, reason, tradeoff, and current owner or status.

✓

Quality gates

Before saving, the agent checks clarity, structure, geometry, evidence, and share polish.

Runtime Flows

The product loop is stable: connect an agent, create a visual, gather feedback, and publish a revised version.

Agent connect

1

Agent discovers OAuth metadata from the MCP endpoint.

2

User signs in and approves the client through FlowThis.

3

Access and refresh tokens map future MCP calls to one account.

Create visual

1

Agent reads `flowthis://visualization-system` if layout is unspecified.

2

Agent submits one complete HTML document with no scripts or external assets.

3

FlowThis stores metadata in D1 and immutable HTML in R2.

Discuss and revise

1

Viewers comment, react, and suggest changes outside the sandboxed content.

2

Agent reads suggestions through MCP and fetches the current HTML.

3

A new version is appended; old versions remain available.

Render safely

1

Access checks decide owner, public, unlisted, invite, share-link, and moderation state.

2

HTML renders in a script-disabled sandbox with strict content CSP.

3

Separate content origin remains the production deployment priority.

Quality Gates

These gates turn "make a visual" into a predictable standard without killing creative layouts.

Clarity

The first screen explains the purpose without needing the original prompt.

Structure

The visual has a reading path: start, map, evidence, action.

Geometry

Diagram arrows stop at boundaries and do not overlap labels or text.

Evidence

Source-backed facts and inferred summaries are visually distinguishable.

Share

Title, provenance, summary, main visual, details, and read-next are present.
