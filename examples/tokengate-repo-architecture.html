TokenGate optimizes retrieved context before it reaches an LLM.

The repository is organized as a typed Python package for RAG context selection: rank, rerank, deduplicate, budget, optionally compress, build the prompt, and audit every decision.

Python 3.12

Package metadata pins Python for ML-wheel reliability.

No LLM inside

README states the optimization pipeline runs offline.

Audit-first

Every included, dropped, or compressed block gets a reason.

MIT

The public repo is open source and app-agnostic.

tokengate/

Public package for context optimization in LLM apps. The README frames it as a replacement for top-k prompt stuffing.

Evidence: README product statement and pipeline section.
Inference: the repo is designed as an embeddable library, not an app-only service.

core/

Owns the optimizer, block model, result objects, and configuration surface.

Evidence: public API exports TokenGate, TokenBlock, OptimizerConfig, OptimizationResult.
Responsibility: keep the high-level optimization contract stable.

ranking/

Combines semantic, keyword, recency, source priority, and reranker signals before selection.

selection/

Adaptive cutoff and MMR diversity decide which candidates remain useful for the final prompt.

deduplication/

Exact, near-duplicate, and semantic dedup reduce repeated context before the budgeter spends tokens.

budgeting/

Token counting and value-per-token fitting keep selected context under the configured prompt budget.

audit/

Stage traces and audit reports explain why blocks were kept, dropped, or compressed.

dashboard/

Optional local FastAPI dashboard visualizes audit history and per-query decisions.
